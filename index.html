PromptDeck

A versioned, parameterized prompt library that turns one-off prompts into reusable team assets.

Teams adopting AI copilots keep rewriting the same prompts and losing the good ones. PromptDeck treats prompts like engineering assets: each template has typed variables, version history, usage notes, and a built-in test runner so anyone can fill in the blanks and get a consistent result.

The seed library is written for consulting workflows such as proposal section drafting, technical report QA, action items from meeting notes, RFP requirement extraction, etc. Instead of just storing prompts, it also focuses on strong prompt design.

What I built

  • Template engine with {{ variable }} substitution, required/optional fields, and inline examples
  • Side-by-side test runner: run a template against the Claude API and compare versions on the same input
  • Version history per prompt with changelog notes
  • Copy and shareable links so non-technical users can adopt templates easily
  • Markdown docs auto-generated from the library, ready to drop into a team wiki

ReportLens

An AI-assisted review workflow for environmental and geotechnical reports.

Environmental consulting runs on long PDFs of Phase I ESAs, geotechnical investigations, and inspection reports. ReportLens ingests a report and produces a structured extraction, an executive summary, and a completeness check against a configurable section checklist, then lets the reviewer ask follow-up questions grounded in the document.

Built and tested on publicly available environmental assessment reports, with an emphasis on trustworthy output: every extracted claim links back to its page of origin.

What I built

  • Python ingestion pipeline that includes PDF parsing, section detection, and chunking tuned for technical reports
  • Structured extraction to JSON (site details, findings, recommendations) with page-level citations
  • Completeness checker that flags missing or thin sections against a report-type checklist
  • Grounded Q&A over the document using embeddings + retrieval
  • Simple Streamlit reviewer UI a non-developer can use without setup instructions

AssetPilot

A tool-calling agent that triages Ontario infrastructure asset data into prioritized and concise briefings.

Municipal asset data is public but underused. AssetPilot is a multi-step, tool-calling agent that pulls Ontario open data on watermain breaks, bridge condition indices, road condition ratings. It runs lightweight Python analysis, and writes a prioritized condition briefing about which assets need attention first, and why, in language a project manager can forward.

The agent plans its own steps (fetch via API, clean, compute risk scores, then draft the briefing) with every number traceable to the analysis layer, so the LLM explains the math instead of inventing it. A scheduled run also emails a weekly digest automatically.

What I built

  • Agent loop with defined tools including a data fetcher, pandas analysis runner, and report writer
  • API integrations with Ontario's municipal open data portals, with caching and retries
  • Risk scoring in Python (age, break history, condition index) which is deterministic, testable, and cited by the agent
  • Automated weekly digest with a scheduled run, generated summary, and email delivery
  • Run logs showing each agent step and tool call, for transparency and debugging
